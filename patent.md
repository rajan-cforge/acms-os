### 1.2 Invention Disclosure Form

**Confidential - Attorney-Client Privileged**

**Date of Disclosure:** October 11, 2025

**Working Title:** Adaptive Context Memory System (ACMS)

**Problem Solved:**
Current AI assistants suffer from context amnesia—they forget user intent, preferences, and past interactions across sessions. Existing solutions either provide no memory (stateless) or store data in the cloud (privacy concerns). No system intelligently learns what information is important based on outcomes and adapts memory retention accordingly.

**Proposed Solution:**
A local-first memory system that:
1. Computes Context Retention Score (CRS) using multi-factor algorithm including outcome success
2. Organizes memory into hierarchical tiers (short/mid/long) with adaptive promotion/demotion
3. Performs predictive rehydration before LLM inference to assemble optimal context bundles
4. Encrypts all data with user-owned keys, ensuring privacy
5. Optionally participates in federated learning without sharing raw data

**Key Innovations:**
1. **Outcome-Aware CRS**: First system to tie memory retention to measured task success
2. **Hierarchical Decay**: Three-tier memory with exponential decay and consolidation
3. **Predictive Rehydration**: Intent-driven context assembly before inference
4. **Local-First Privacy**: Hardware-backed encryption with user-owned keys
5. **Token Optimization**: 30-50% reduction in LLM inference costs

**Competitive Advantages:**
- No prior art combines all five innovations
- Patent-defensible algorithm (CRS formula is novel)
- First-mover in regulated industries (HIPAA/GDPR compliant by design)

**Commercial Applications:**
- Enterprise AI assistants (security, healthcare, finance, legal)
- Personal AI coaches and productivity tools
- Developer SDKs for AI memory integration

**Status of Reduction to Practice:**
- Conceptual design: Complete
- Proof of concept: In progress (MVP planned)
- Working prototype: Target Week 9

**Prior Disclosure:**
- None (maintained strict confidentiality)
- No publications, presentations, or demonstrations to third parties

**Related Work:**

## 4. Claims Strengthening Analysis

### 4.1 Claim Construction Review

**Independent Claim 1 (Method):**

**Original:**
> A computer-implemented method for adaptive context memory in an AI assistant executed on a user device, comprising...

**Potential Weaknesses:**
1. "User device" could be construed narrowly (only laptops/phones)
2. "AI assistant" might be too broad or vague

**Strengthened Version:**
> A computer-implemented method for adaptive context memory in an artificial intelligence system executing on a computing device under user control, comprising...

**Rationale:**
- "Computing device under user control" covers servers, workstations, edge devices
- "Artificial intelligence system" more precise, includes chatbots, agents, assistants

---

**CRS Computation Element:**

**Original:**
> computing, for each item, a context retention score (CRS) based on semantic relevance, recurrence frequency, outcome success correlation, user correction signals, and temporal decay

**Potential Weaknesses:**
- Could be interpreted as requiring all five factors (overly limiting)

**Strengthened Version:**
> computing, for each item, a context retention score (CRS) by: (i) calculating a plurality of relevance factors including at least semantic relevance, recurrence frequency, and outcome success correlation; (ii) applying a temporal decay function based on item age; and (iii) combining said factors into a bounded score

**Rationale:**
- "At least" allows for flexibility in factor selection
- Separates temporal decay as distinct step (harder to design around)
- "Bounded score" (e.g., [0,1]) is a specific technical feature

---

### 4.2 Dependent Claims Strategy

**Defensive Claims (Prevent Design-Arounds):**

**Claim X:** The method of claim 1, wherein the temporal decay function is exponential with a configurable decay rate parameter λ.

**Rationale:** Specifies exponential decay (vs. linear, polynomial), making it harder to design around.

**Claim Y:** The method of claim 1, wherein the outcome success correlation is computed from at least one of: edit distance reduction, explicit user approval signals, or task completion indicators.

**Rationale:** Enumerates specific outcome signals, creating multiple layers of protection.

---

**Reach Claims (Capture Variations):**

**Claim Z:** The method of claim 1, wherein the hierarchical memory tiers comprise at least two tiers selected from: a short-term tier with retention measured in hours, a mid-term tier with retention measured in days, and a long-term tier with retention measured in months.

**Rationale:** "At least two tiers" covers simpler implementations (only S+L) while still describing hierarchical structure.

---

**Commercial Claims (Cover Likely Embodiments):**

**Claim W:** The method of claim 1, further comprising: integrating with a large language model via an application programming interface; and injecting the summarized context into a prompt template prior to inference.

**Rationale:** Covers the most common commercial use case (LLM API integration).

## 5. Patent Prosecution Strategy

### 5.1 Filing Strategy

**Option A: Provisional Application (Recommended)**

**Timeline:**
- Week 3: File provisional ($150 USPTO fee + $3-5K attorney)
- Month 12: File full utility patent ($1,780 base + $15-20K attorney)
- Month 18-36: Prosecution (office actions, responses)
- Month 24-48: Grant (if approved)

**Advantages:**
- Establishes early priority date
- Lower upfront cost
- 12 months to refine claims based on MVP learnings
- Can add more embodiments discovered during development

**Disadvantages:**
- No examination during provisional year
- Must file full application within 12 months or lose priority

---

**Option B: Full Utility Application**

**Timeline:**
- Week 3: File full utility patent ($1,780 base + $15-25K attorney)
- Month 12-24: First office action
- Month 24-36: Final action or allowance
- Month 30-48: Grant

**Advantages:**
- Examination begins immediately
- No need for conversion
- Earlier grant date (if successful)

**Disadvantages:**
- Higher upfront cost
- Less flexibility to refine claims
- Must have complete invention description at filing

**Recommendation:** Option A (Provisional) to allow MVP validation and claim refinement.

### 5.3 Prosecution Tactics

**Anticipate Office Actions:**

**Likely Rejection #1:** 35 USC 102 (anticipation) citing MemGPT or similar

**Response Strategy:**
- Point to outcome-aware CRS (not in MemGPT)
- Emphasize local-first encryption (not in MemGPT)
- Highlight predictive rehydration with intent (not in MemGPT)
- Argue combination is novel

---

**Likely Rejection #2:** 35 USC 103 (obviousness) - combination of RAG + tiered memory

**Response Strategy:**
- Show synergistic effect (outcome learning → 20% relevance boost)
- Demonstrate unexpected results (30-50% token savings)
- Provide test data from MVP
- Argue no motivation to combine in this specific way
- Commercial success evidence (if available)

---

**Likely Rejection #3:** 35 USC 101 (subject matter eligibility) - abstract idea

**Response Strategy:**
- Emphasize technical improvements (token reduction, latency optimization)
- Cite specific hardware (TPM, Secure Enclave)
- Show unconventional computer implementation
- Demonstrate specific improvement to computer functionality
- Use "significantly more" framework (Alice/Mayo test)

**Precedents to Cite:**
- *DDR Holdings v. Hotels.com* (improved computer functionality)
- *Enfish v. Microsoft* (self-referential database = patent-eligible)
- *McRO v. Bandai Namco* (specific implementation of rules = patent-eligible)
